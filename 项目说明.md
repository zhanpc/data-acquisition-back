# 数据采集系统

基于 IEC 60870-5-104 和 Modbus 协议的工业数据采集系统，支持大规模场站接入和海量时序数据存储。

## 系统架构

```
采集层 (IEC104/Modbus)
    ↓
配置缓存 (内存)
    ↓
数据生产者 (WAL + Kafka)
    ↓
Kafka 消息队列
    ↓
数据消费者 (批量消费)
    ↓
TDengine 时序数据库
    ↓
REST API (查询/配置/控制)
```

## 核心特性

✅ **数据不丢失**: WAL + Kafka + 手动提交三层保障  
✅ **高性能**: 内存缓存 + 批量写入 + 时序数据库  
✅ **可扩展**: 从 2 个场站平滑扩展到 200 个场站  
✅ **低延迟**: 端到端延迟 <30ms  
✅ **易运维**: Docker 部署 + 完善监控

## 快速开始

### 1. 环境准备

**手动安装（持久化数据）:**
- TDengine 3.x
- MySQL 8.0

**Docker 部署:**
```bash
docker-compose up -d
```

### 2. 初始化数据库

```bash
# MySQL
mysql -u root -p < sql/mysql_init.sql

# TDengine
taos < sql/tdengine_init.sql
```

### 3. 编译运行

```bash
# 编译
mvn clean package -DskipTests

# 运行
java -jar target/app.jar --spring.profiles.active=test
```

### 4. 访问 API 文档

浏览器访问: `http://localhost:8005/test`

## 数据流程

### 采集流程

1. **设备连接**: IEC104/Modbus 设备连接
2. **数据解析**: 协议解析，提取测点数据
3. **配置映射**: 从内存缓存获取测点配置
4. **数据转换**: 构造 DataPoint 对象
5. **WAL 备份**: 写入本地 WAL 文件
6. **Kafka 发送**: 发送到 Kafka 消息队列
7. **批量消费**: Consumer 批量拉取数据
8. **持久化**: 批量写入 TDengine
9. **提交 Offset**: 手动提交 Kafka offset

### 配置管理

- **初始化**: 应用启动时从 MySQL 加载配置到内存
- **运行时**: 采集服务仅读取内存缓存
- **变更**: 修改 MySQL 后调用刷新接口重新加载

## 数据不丢失保障

### 三层保障机制

1. **WAL 本地备份**
   - 数据先写入本地文件
   - 发送成功后删除
   - 进程崩溃可恢复

2. **Kafka 持久化**
   - acks=1（至少一个副本确认）
   - 重试机制（最多10次）
   - 幂等性保证

3. **手动提交 Offset**
   - 写入 TDengine 成功后才提交
   - 失败自动重试
   - 避免数据丢失

## API 接口

### 配置管理

- `POST /api/station/config` - 创建场站配置
- `GET /api/station/config` - 查询场站列表
- `POST /api/station/config/refresh` - 刷新配置缓存

### 数据查询

- `GET /api/data/latest` - 查询实时数据
- `GET /api/data/history` - 查询历史曲线
- `GET /api/data/aggregate` - 查询统计数据

### 采集控制

- `POST /api/collection/start/{stationId}` - 启动采集
- `POST /api/collection/stop/{stationId}` - 停止采集
- `GET /api/collection/status` - 查询状态

## 监控告警

### 监控指标

- 采集速率: 实际点数/秒 vs 预期点数/秒
- Kafka 延迟: Consumer lag
- TDengine 写入速率: 点/秒
- 系统资源: CPU、内存、磁盘

### 告警规则

- 场站离线: 紧急
- 采集速率下降 >20%: 警告
- Kafka 消费延迟 >10000条: 严重
- 磁盘使用率 >85%: 警告

## 故障排查

### 常见问题

1. **应用无法启动**
   - 检查 Java 版本
   - 检查端口占用
   - 查看日志文件

2. **Kafka 连接失败**
   - 检查 Docker 服务状态
   - 验证网络连接
   - 查看 Kafka 日志

3. **TDengine 连接失败**
   - 检查服务状态
   - 验证数据库是否存在
   - 检查端口开放

4. **数据未写入**
   - 检查 Kafka 消费者 lag
   - 查看应用日志
   - 验证表结构
