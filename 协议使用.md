# 数据采集协议使用说明

本项目实现了 Modbus TCP/RTU 和 IEC 60870-5-104 协议的数据采集功能。

## 依赖说明

已在 `pom.xml` 中添加以下依赖：
- `modbus4j`: Modbus TCP/RTU 协议实现
- `j60870`: IEC 60870-5-104 协议实现
- `rxtx`: 串口通信库（用于 Modbus RTU）

## 服务类说明

### 1. ModbusTcpService
Modbus TCP 协议数据采集服务

**主要方法：**
- `connect(connectionId, host, port)`: 连接设备
- `readHoldingRegisters(connectionId, slaveId, offset, length)`: 读取保持寄存器
- `readInputRegisters(connectionId, slaveId, offset, length)`: 读取输入寄存器
- `readCoils(connectionId, slaveId, offset, length)`: 读取线圈状态
- `writeSingleRegister(connectionId, slaveId, offset, value)`: 写单个寄存器
- `writeSingleCoil(connectionId, slaveId, offset, value)`: 写单个线圈
- `disconnect(connectionId)`: 断开连接

### 2. ModbusRtuService
Modbus RTU 协议数据采集服务（通过串口通信）

**主要方法：**
- `connect(connectionId, portName, baudRate, dataBits, stopBits, parity)`: 连接设备
- `readHoldingRegisters(...)`: 读取保持寄存器
- `writeSingleRegister(...)`: 写单个寄存器
- `disconnect(connectionId)`: 断开连接

### 3. Iec104Service
IEC 60870-5-104 协议数据采集服务

**主要方法：**
- `connect(connectionId, host, port)`: 连接设备
- `generalInterrogation(connectionId, commonAddress)`: 总召唤
- `readSinglePoint(connectionId, commonAddress, ioa)`: 读取单点信息
- `readShortFloat(connectionId, commonAddress, ioa)`: 读取浮点数
- `sendSingleCommand(connectionId, commonAddress, ioa, value)`: 发送单点命令
- `sendSetpointCommand(connectionId, commonAddress, ioa, value)`: 发送设定值命令
- `disconnect(connectionId)`: 断开连接

### 4. DataAcquisitionService
数据采集管理服务，统一管理所有协议连接

## 使用示例

### Modbus TCP 示例

```java
@Autowired
private ModbusTcpService modbusTcpService;

// 连接设备
String connectionId = "device-001";
modbusTcpService.connect(connectionId, "192.168.1.100", 502);

// 读取数据
int[] values = modbusTcpService.readHoldingRegisters(connectionId, 1, 0, 10);

// 写入数据
modbusTcpService.writeSingleRegister(connectionId, 1, 0, 1234);

// 断开连接
modbusTcpService.disconnect(connectionId);
```

### IEC 104 示例

```java
@Autowired
private Iec104Service iec104Service;

// 连接设备
String connectionId = "iec104-001";
iec104Service.connect(connectionId, "192.168.1.200", 2404);

// 总召唤
iec104Service.generalInterrogation(connectionId, 1);

// 读取数据
Boolean value = iec104Service.readSinglePoint(connectionId, 1, 100);
Float floatValue = iec104Service.readShortFloat(connectionId, 1, 200);

// 发送命令
iec104Service.sendSingleCommand(connectionId, 1, 100, true);

// 断开连接
iec104Service.disconnect(connectionId);
```

## 运行测试

修改测试类中的 IP 地址和端口为实际设备地址，然后运行测试：

```bash
# 运行 Modbus TCP 测试
mvn test -Dtest=ModbusTcpServiceTest

# 运行 IEC 104 测试
mvn test -Dtest=Iec104ServiceTest
```

## 注意事项

1. **Modbus TCP**: 默认端口 502，需要确保设备支持 Modbus TCP 协议
2. **Modbus RTU**: 需要配置正确的串口参数（波特率、数据位、停止位、校验位）
3. **IEC 104**: 默认端口 2404，需要了解设备的公共地址(COA)和信息对象地址(IOA)
4. 测试前请确保设备在线且网络可达
5. 生产环境使用时建议添加异常处理和重连机制

## 协议参数说明

### Modbus 参数
- **slaveId**: 从站地址（1-247）
- **offset**: 寄存器起始地址
- **length**: 读取长度

### IEC 104 参数
- **commonAddress**: 公共地址（COA）
- **ioa**: 信息对象地址（IOA）
- **TypeID**: 类型标识（由协议自动处理）
