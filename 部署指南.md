# 数据采集系统部署文档

## 系统要求

### 硬件要求
- **CPU**: 16核（国产化CPU）
- **内存**: 128GB
- **存储**: 
  - 2×1TB SSD (RAID1) - 热数据存储
  - 2×8TB HDD (RAID5) - 冷数据归档
- **网络**: 6个千兆电口，2个千兆光口

### 软件要求
- **操作系统**: Linux (CentOS 7/8, Ubuntu 18.04+)
- **Java**: JDK 1.8+
- **Docker**: 20.10+
- **Docker Compose**: 1.29+
- **Maven**: 3.6+

## 部署步骤

### 1. 安装 TDengine（手动安装）

TDengine 作为时序数据库，需要手动安装以确保数据持久化。

```bash
# 下载 TDengine 3.x
wget https://www.taosdata.com/assets-download/3.0/TDengine-server-3.2.5.0-Linux-x64.tar.gz

# 解压
tar -zxvf TDengine-server-3.2.5.0-Linux-x64.tar.gz
cd TDengine-server-3.2.5.0

# 安装
sudo ./install.sh

# 启动服务
sudo systemctl start taosd
sudo systemctl enable taosd

# 验证安装
taos
```

#### 配置 TDengine

编辑 `/etc/taos/taos.cfg`：

```ini
# 数据目录（SSD）
dataDir /data/tdengine

# 日志目录
logDir /var/log/taos

# 最大虚拟节点组数
maxVgroupsPerDb 64

# 每个虚拟节点最大表数
maxTablesPerVnode 1000000

# 缓存大小（MB）
cache 256

# 数据块大小（MB）
blocks 6
```

重启服务：
```bash
sudo systemctl restart taosd
```

#### 初始化数据库

```bash
# 登录 TDengine
taos

# 执行初始化脚本
source /path/to/project/sql/tdengine_init.sql
```

### 2. 安装 MySQL（手动安装）

MySQL 用于存储配置数据，需要手动安装。

```bash
# CentOS/RHEL
sudo yum install mysql-server

# Ubuntu/Debian
sudo apt-get install mysql-server

# 启动服务
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时密码（CentOS）
sudo grep 'temporary password' /var/log/mysqld.log

# 安全配置
sudo mysql_secure_installation
```

#### 初始化数据库

```bash
# 登录 MySQL
mysql -u root -p

# 执行初始化脚本
source /path/to/project/sql/mysql_init.sql
```

### 3. 部署 Kafka 和 Redis（Docker Compose）

Kafka 和 Redis 使用 Docker 部署，便于管理。

```bash
# 进入项目目录
cd /path/to/data-acquisition-back

# 创建数据目录
sudo mkdir -p /data/zookeeper
sudo mkdir -p /data/kafka
sudo mkdir -p /data/redis
sudo mkdir -p /data/wal

# 启动 Docker Compose
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f kafka
```

#### 验证 Kafka

```bash
# 进入 Kafka 容器
docker exec -it iot-kafka bash

# 创建测试 Topic
kafka-topics.sh --create --topic test --bootstrap-server localhost:9092 --partitions 10 --replication-factor 1

# 查看 Topic 列表
kafka-topics.sh --list --bootstrap-server localhost:9092

# 删除测试 Topic
kafka-topics.sh --delete --topic test --bootstrap-server localhost:9092
```

### 5. 配置应用

编辑 `src/main/resources/config/application-prod.yml`（如果没有则创建）：

```yaml
server:
  port: 8005

spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://127.0.0.1:3306/iot_config?serverTimezone=Asia/Shanghai&characterEncoding=utf-8&useSSL=false&useUnicode=true
    username: root
    password: your_mysql_password
  
  kafka:
    bootstrap-servers: localhost:9092
    producer:
      acks: 1
      retries: 10
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
      properties:
        enable.idempotence: true
        max.in.flight.requests.per.connection: 5
    consumer:
      group-id: iot-data-consumer
      auto-offset-reset: earliest
      enable-auto-commit: false
      max-poll-records: 1000
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.springframework.kafka.support.serializer.JsonDeserializer
      properties:
        spring.json.trusted.packages: com.maplestone.dataCollect.pojo.entity

tdengine:
  datasource:
    driver-class-name: com.taosdata.jdbc.TSDBDriver
    url: jdbc:TAOS://127.0.0.1:6030/iot_data?charset=UTF-8&locale=en_US.UTF-8&timezone=UTC-8
    username: root
    password: taosdata

redis:
  host: 127.0.0.1
  port: 6379
  timeout: 3600
  password:
  database: 0
  pool:
    maxTotal: 1000
    maxIdle: 100
    maxWaitMillis: 100000
    minIdle: 80

wal:
  enabled: true
  base-path: /data/wal
  max-file-size: 10485760
  retention-hours: 24
```

## 故障排查

### 应用无法启动

1. 检查 Java 版本：`java -version`
2. 检查端口占用：`netstat -tunlp | grep 8005`
3. 查看日志：`tail -f app.log`
4. 检查配置文件路径和内容

### Kafka 连接失败

1. 检查 Kafka 服务状态：`docker-compose ps`
2. 查看 Kafka 日志：`docker-compose logs kafka`
3. 验证网络连接：`telnet localhost 9092`
4. 检查防火墙规则

### TDengine 连接失败

1. 检查服务状态：`sudo systemctl status taosd`
2. 查看日志：`tail -f /var/log/taos/taosd.log`
3. 验证端口：`netstat -tunlp | grep 6030`
4. 检查数据库是否存在：`taos -s "SHOW DATABASES;"`

### MySQL 连接失败

1. 检查服务状态：`sudo systemctl status mysqld`
2. 验证用户权限：`mysql -u root -p`
3. 检查数据库是否存在：`SHOW DATABASES;`

### 数据未写入 TDengine

1. 检查 Kafka 消费者 lag：
   ```bash
   docker exec -it iot-kafka kafka-consumer-groups.sh --bootstrap-server localhost:9092 --group iot-data-consumer --describe
   ```
2. 查看应用日志中的错误信息
3. 验证 TDengine 表结构是否正确
4. 检查 WAL 目录中是否有未发送的数据

## 性能优化

### JVM 参数优化

```bash
java -Xms8g -Xmx8g \
     -XX:+UseG1GC \
     -XX:MaxGCPauseMillis=200 \
     -XX:+HeapDumpOnOutOfMemoryError \
     -XX:HeapDumpPath=/data/heapdump \
     -jar target/app.jar --spring.profiles.active=prod
```

### TDengine 参数优化

编辑 `/etc/taos/taos.cfg`：

```ini
# 增加缓存
cache 512

# 增加数据块
blocks 12

# 启用压缩
comp 2
```

### Kafka 参数优化

编辑 `docker-compose.yml`：

```yaml
environment:
  KAFKA_NUM_PARTITIONS: 10
  KAFKA_LOG_SEGMENT_BYTES: 1073741824
  KAFKA_LOG_RETENTION_CHECK_INTERVAL_MS: 300000
```

### 关键指标

- **采集速率**: 实际点数/秒 vs 预期点数/秒

- **TDengine 写入速率**: 点/秒

## 备份与恢复

### TDengine 备份

```bash
# 停止写入
# 备份数据目录
sudo tar -czf tdengine-backup-$(date +%Y%m%d).tar.gz /data/tdengine

# 恢复
sudo tar -xzf tdengine-backup-20260316.tar.gz -C /
sudo systemctl restart taosd
```

### MySQL 备份

```bash
# 备份
mysqldump -u root -p iot_config > iot_config-backup-$(date +%Y%m%d).sql

# 恢复
mysql -u root -p iot_config < iot_config-backup-20260316.sql
```

## 安全建议

1. **修改默认密码**: MySQL、TDengine、Redis
2. **启用防火墙**: 仅开放必要端口
3. **配置 SSL/TLS**: 生产环境启用加密通信
4. **限制网络访问**: 数据库仅允许本地访问
5. **定期更新**: 及时更新系统和依赖包
6. **日志审计**: 启用操作日志记录
