# 数据采集系统架构设计文档（增强版）

## 项目概述

基于IEC 60870-5-104、Modbus和OPC协议的工业数据采集系统，支持大规模场站接入和海量时序数据存储。系统支持50万个采集点的数据采集、存储和实时处理，具备完整的数据不丢失保障机制和边缘数据处理能力。

### 核心能力

- **多协议支持**: IEC 60870-5-104、Modbus TCP/RTU、OPC UA/DA
- **大规模采集**: 支持50万个采集点，200个场站
- **数据不丢失**: WAL + Kafka + 手动提交三层保障，30天离线数据缓存
- **实时处理**: 边缘数据处理、阈值告警、数据质量校验
- **断点续传**: 双通道设计（实时通道 + 历史补传通道）
- **远程控制**: 支持遥控命令下发、执行反馈和审计
- **统一配置**: Web界面统一配置协议参数和采集频率（秒级可配）
- **多通道融合**: 多数据源融合转发站架构

### 硬件配置
- **CPU**: 国产化CPU，16核
- **内存**: 128GB
- **存储**:
  - 2×1TB SSD (RAID1) - 热数据存储
  - 2×8TB HDD (RAID5) - 冷数据归档
- **网络**: 6个千兆电口，2个千兆光口
- **电源**: 冗余电源
- **管理**: 1个IPMI管理网口

### 数据规模
- **测试阶段**: 2个场站，2TB/年
- **扩展目标**: 20个场站，20TB/年
- **最终目标**: 200个场站，200TB/年，50万个采集点

## 协议支持

### 支持的协议

系统支持三种主流工业控制协议，满足不同场景的数据采集需求：

#### 1. IEC 60870-5-104 协议

**适用场景**: 电力系统SCADA、变电站自动化、配电自动化

**技术特性**:
- 基于TCP/IP的应用层协议
- 支持总召唤、时钟同步、遥控命令
- APDU/ASDU报文封装
- 支持遥测、遥信、遥控、遥调四遥功能

**数据点类型**:
- 遥测（模拟量）: M_ME_NC_1 (类型13) - 浮点数测量值
- 遥信（数字量）: M_SP_NA_1 (类型1) - 单点信息
- 遥控: C_SC_NA_1 (类型45) - 单命令
- 遥调: C_SE_NC_1 (类型50) - 设定值命令

**采集点容量**: 单场站支持10,000个采集点

#### 2. Modbus TCP/RTU 协议

**适用场景**: 工业自动化、PLC通信、传感器数据采集

**技术特性**:
- Modbus TCP: 基于TCP/IP，端口502
- Modbus RTU: 基于RS485串口通信
- 支持功能码: 01/02/03/04/05/06/15/16
- 寄存器类型: 线圈、离散输入、保持寄存器、输入寄存器

**数据点类型**:
- 模拟量: 保持寄存器(40001-49999)、输入寄存器(30001-39999)
- 数字量: 线圈(00001-09999)、离散输入(10001-19999)

**采集点容量**: 单场站支持5,000个采集点

#### 3. OPC UA/DA 协议

**适用场景**: 企业级数据集成、跨平台数据交换、工业物联网

**技术特性**:
- OPC UA: 基于TCP/IP，支持安全认证和加密
- OPC DA: 基于DCOM，Windows平台
- 支持订阅机制、历史数据访问
- 节点浏览和数据类型发现

**数据点类型**:
- 模拟量: Float, Double, Int32, UInt32
- 数字量: Boolean
- 字符串: String
- 复杂类型: Structure, Array

**采集点容量**: 单场站支持20,000个采集点

### 协议对比

| 特性 | IEC 60870-5-104 | Modbus TCP/RTU | OPC UA/DA |
|------|----------------|----------------|-----------|
| 传输层 | TCP/IP | TCP/IP / RS485 | TCP/IP / DCOM |
| 应用场景 | 电力SCADA | 工业自动化 | 企业集成 |
| 数据类型 | 遥测/遥信/遥控/遥调 | 寄存器/线圈 | 丰富类型 |
| 安全性 | 中 | 低 | 高 |
| 复杂度 | 高 | 低 | 高 |
| 单场站容量 | 10,000点 | 5,000点 | 20,000点 |
| 实时性 | 高 | 中 | 高 |

### 系统总容量

- **总采集点数**: 500,000个
- **场站数量**: 200个
- **平均每场站**: 2,500个采集点
- **峰值场站**: 20,000个采集点（OPC协议）

## 数据点分类

### 数据点类型定义

系统支持四种标准数据点类型，对应工业控制系统的"四遥"功能：

#### 1. 遥测（模拟量 - Analog）

**定义**: 连续变化的物理量测量值

**典型数据**:
- 电压、电流、功率
- 温度、压力、流量
- 频率、相位角
- 液位、转速

**数据类型**: Float, Double, Int32
**采集频率**: 1秒 ~ 60秒可配
**存储精度**: 保留4位小数

#### 2. 遥信（数字量 - Digital）

**定义**: 离散状态量，表示开关状态或事件

**典型数据**:
- 断路器状态（分/合）
- 设备运行状态（启动/停止）
- 告警信号（正常/告警）
- 门禁状态（开/关）

**数据类型**: Boolean, Byte
**采集频率**: 实时变位上送 + 定时轮询（5秒）
**存储方式**: 变位存储（仅存储状态变化）

#### 3. 遥控（Remote Control）

**定义**: 远程控制命令，用于操作设备

**典型操作**:
- 断路器分合闸
- 设备启停控制
- 阀门开关控制
- 参数设置

**命令类型**:
- 单命令（Select-Execute）
- 双命令（On/Off）
- 设定值命令

**执行流程**: 选择 → 执行 → 反馈 → 审计

#### 4. 遥调（Remote Adjustment）

**定义**: 远程参数调节，用于设定设备运行参数

**典型操作**:
- 电压设定值调整
- 功率因数设定
- PID参数调整
- 温度设定值

**数据类型**: Float, Int32
**执行流程**: 参数验证 → 下发 → 确认 → 记录

### 协议参数配置

#### IEC 60870-5-104 配置项

```java
public class IEC104Config {
    // 连接参数
    private String host;                // 服务器地址
    private Integer port;               // 端口（默认2404）
    private Integer coa;                // 公共地址

    // 采集参数
    private Integer collectFreq;        // 采集频率（秒）1-3600
    private Integer generalCallInterval;// 总召唤间隔（秒）300-86400
    private Integer clockSyncInterval;  // 时钟同步间隔（秒）3600

    // 通信参数
    private Integer timeout;            // 超时时间（毫秒）1000-30000
    private Integer retryCount;         // 重试次数 1-10
    private Integer retryInterval;      // 重试间隔（毫秒）100-5000

    // 心跳参数
    private Integer heartbeatInterval;  // 心跳间隔（秒）10-300
    private Integer heartbeatTimeout;   // 心跳超时（秒）30-600

    // APDU参数
    private Integer k;                  // 发送窗口大小 1-32767
    private Integer w;                  // 接收窗口大小 1-32767
    private Integer t0;                 // 连接超时（秒）10-255
    private Integer t1;                 // 发送确认超时（秒）5-255
    private Integer t2;                 // 接收确认超时（秒）5-255
    private Integer t3;                 // 测试帧超时（秒）10-172800
}
```

#### Modbus配置项

public class ModbusConfig {
    // 连接参数
    private String protocol;            // TCP/RTU
    private String host;                // TCP地址
    private Integer port;               // TCP端口（默认502）
    private String serialPort;          // RTU串口（如COM1）
    private Integer baudRate;           // RTU波特率（9600/19200/38400）
    private Integer dataBits;           // 数据位（7/8）
    private Integer stopBits;           // 停止位（1/2）
    private String parity;              // 校验位（NONE/ODD/EVEN）

    // 采集参数
    private Integer slaveId;            // 从站地址 1-247
    private Integer collectFreq;        // 采集频率（秒）1-3600
    private Integer batchSize;          // 批量读取数量 1-125

### 断点续传流程

```
正常运行:
1. 数据采集 → 实时通道 → 立即发送
2. 数据成功写入TDengine
3. 更新最后成功时间戳

网络中断:
1. 检测到网络故障
2. 数据写入本地WAL（30天容量）
3. 标记为待补传数据
4. 继续采集新数据到WAL

网络恢复:
1. 检测到网络恢复
2. 查询最后成功时间戳
3. 从WAL读取断点后的数据
4. 数据进入历史通道
5. 后台批量补传（不影响实时数据）
6. 补传完成后清理WAL

补传策略:
- 实时通道: 100条/批，延迟<100ms
- 历史通道: 10000条/批，延迟<5秒
- 历史通道限速: 最大50MB/s（避免影响实时通道）
- 补传进度监控: 实时显示补传进度
```

#### WAL文件管理

```java
public class WALManager {
    // WAL配置
    private String walPath = "/data/wal";           // WAL目录
    private Long maxFileSize = 100 * 1024 * 1024L; // 单文件100MB
    private Integer retentionDays = 30;             // 保留30天
    private String compressionType = "LZ4";         // 压缩算法

    // WAL文件命名: wal_stationId_timestamp.dat
    // 例如: wal_001_1710662400000.dat

    // 写入流程
    public void write(DataPoint data) {
        // 1. 序列化数据
        byte[] bytes = serialize(data);

        // 2. 写入当前WAL文件
        currentFile.write(bytes);

        // 3. 检查文件大小
        if (currentFile.size() >= maxFileSize) {
            // 4. 关闭当前文件
            currentFile.close();

            // 5. 压缩文件（异步）
            compressAsync(currentFile);

            // 6. 移动到HDD（如果超过24小时）
            if (currentFile.age() > 24 * 3600) {
                moveToArchive(currentFile);
            }

            // 7. 创建新文件
            currentFile = createNewFile();
        }
    }

    // 读取流程（断点续传）
    public List<DataPoint> read(Long startTs, Long endTs) {
        // 1. 查找时间范围内的WAL文件
        List<File> files = findFiles(startTs, endTs);

        // 2. 按时间顺序读取
        List<DataPoint> result = new ArrayList<>();
        for (File file : files) {
            // 3. 解压缩（如果需要）
            if (file.isCompressed()) {
                file = decompress(file);
            }

            // 4. 反序列化数据
            List<DataPoint> data = deserialize(file);

            // 5. 过滤时间范围
            data = data.stream()
                .filter(d -> d.getTimestamp() >= startTs && d.getTimestamp() <= endTs)
                .collect(Collectors.toList());

            result.addAll(data);
        }

        return result;
    }

    // 清理流程
    public void cleanup() {
        // 1. 查找过期文件（>30天）
        List<File> expiredFiles = findExpiredFiles(retentionDays);

        // 2. 删除过期文件
        for (File file : expiredFiles) {
            file.delete();
        }

        // 3. 记录清理日志
        log.info("Cleaned up {} expired WAL files", expiredFiles.size());
    }
}
```

#### 缓存容量计算

```
假设:
- 200个场站
- 每场站2500个采集点
- 总计50万个采集点
- 采集频率: 平均10秒/次
- 单条数据: 200字节

每秒数据量:
50万点 / 10秒 = 5万条/秒
5万条 × 200字节 = 10MB/秒

每天数据量:
10MB/秒 × 86400秒 = 864GB/天

30天数据量:
864GB × 30 = 25.9TB

压缩后（5:1）:
25.9TB / 5 = 5.2TB

实际配置:
- SSD: 100GB（24小时）
- HDD: 2TB（30天压缩数据）
- 冗余: 2倍（RAID5）
- 总需求: 4TB HDD
```
